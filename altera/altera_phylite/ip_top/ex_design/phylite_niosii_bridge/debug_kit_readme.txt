###################################################################
# Instructions for generating the Nios 2 Instruction Memory
###################################################################

1. Run: qsys-edit phylite_debug_kit.qsys
2. Select the desired Arria 10 device from "Device Family" tab. 
3. Click on "Generate HDL" on the bottom right to generate the project. This will also generate phylite_debug_kit.sopcinfo
4. Create a new Quartus project: File -> New Project Wizard
5. Specify phylite_debug_kit as project name
6. Add phylite_debug_kit.qip into the project 
7. From Quartus: select Tools -> Nios II Software Build Tools for Eclipse
8. Create a new workspace if prompted
9. Select File -> New -> Nios II Application and BSP from Template
10. Select the phylite_debug_kit.sopcinfo file during generation. Select that file for the "SOPC Information File name" field. Once selected Eclipse will try to automatically load all the CPU settings (e.g. address space info)
11. Specify desired project name. E.g. phylite_test
12. Specify "Hello World" as the "Project Template"
13. Click Finish to generate project
14. Add phylite_dynamic_reconfiguration.c and phylite_dynamic_reconfiguration.h source files to the project (these will be created as empty templates)
15. Copy the hello_world.c, phylite_dynamic_reconfiguration.c, and phylite_dynamic_reconfiguration.h generated by the debug example design into the Eclipse project
16. Open all 3 source files in eclipse and hit F5 to refresh
17. Select Project -> Build All, make sure there is no error. This will generate a .elf file under your software directory.
18. Open up NiosII Command Shell (Command PromptA) and convert .elf to a HEX via elf2hex and loaded as the memory initialization file for the instruction ROM:
	elf2hex --input=<elf file name>.elf --base=0x40000 --end=0x7ffff --width=32 --output=phylite_debug_kit_inst_mem.hex
19. Copy this file into your Quartus project directory and add the hex file to your Quartus project:
    set_global_assignment -name MISC_FILE phylite_debug_kit_inst_mem.hex
20. Compile the quartus project
21. Download the sof into the FPGA
22. Open up a new  NiosII Command Shell (Command PromptB) and use the following command to run the Nios II terminal application for result printouts.
    nios2-terminal --cable=<jtag_cable_num>
23. In Command PromptA, run the following command to reset the system and start the NIOS program run.
    quartus_stp -t issp.tcl phylite_debug_kit.qpf

Misc commands:
# To update the instruction memory on a sof running on the FPGA:
nios2-download --cable=<jtag_cable-num> -r <elf file name>.elf
# To view UART output
nios2-terminal --cable=<jtag_cable_num>
